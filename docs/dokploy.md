# Dokploy

Self-hosted PaaS (Vercel/Netlify-style) running on Docker Swarm. The role mirrors the upstream `install.sh` as idempotent Ansible tasks using the `community.docker` collection.

## Configuration

### Default Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `dokploy_version` | `latest` | Dokploy image tag |
| `dokploy_image` | `dokploy/dokploy` | Dokploy image |
| `dokploy_postgres_image` | `postgres:16` | Postgres image |
| `dokploy_redis_image` | `redis:7` | Redis image |
| `dokploy_traefik_image` | `traefik:v3.6.7` | Traefik image |
| `dokploy_data_dir` | `/etc/dokploy` | Host config / state directory |
| `dokploy_postgres_password_file` | `/etc/dokploy/.postgres_password` | On-host source-of-truth for the generated postgres password |
| `dokploy_network_name` | `dokploy-network` | Overlay network name |
| `dokploy_advertise_addr` | `{{ ansible_default_ipv4.address }}` | Address used by `docker swarm init` |
| `dokploy_app_port` | `3000` | Dokploy admin UI host port |
| `dokploy_http_port` | `80` | Traefik HTTP host port |
| `dokploy_https_port` | `443` | Traefik HTTPS host port |

## Ports

| Port | Description |
|------|-------------|
| 3000 | Dokploy admin UI |
| 80   | Traefik (HTTP, routes deployed apps) |
| 443  | Traefik (HTTPS, routes deployed apps) |

All three are LAN-only listeners — they are *not* publicly reachable unless the router forwards those ports. See [Cloudflare tunnel integration](#cloudflare-tunnel-integration) below.

## File Locations

| Path (on host) | Description |
|----------------|-------------|
| `/etc/dokploy/` | Dokploy config + Traefik configs (mode 0777, matches upstream) |
| `/etc/dokploy/traefik/traefik.yml` | Generated by Dokploy on first start |
| `/etc/dokploy/traefik/dynamic/` | Dynamic Traefik middleware/router configs (managed by Dokploy) |
| `/etc/dokploy/.postgres_password` | Generated random password (mode 0600), reused on subsequent runs |
| Docker volume `dokploy-postgres` | Postgres data |
| Docker volume `dokploy-redis` | Redis data |
| Docker volume `dokploy` | Dokploy's `/root/.docker` (registry credentials etc.) |
| Docker secret `dokploy_postgres_password` | Postgres credential injected at `/run/secrets/postgres_password` |

## Service Details

- **Runtime**: Docker Swarm (single-manager). The first Swarm-mode service in this repo.
- **Postgres / Redis / Dokploy**: managed as `docker service`s (rolling updates via `docker service update`).
- **Traefik**: a standalone `docker run` container (matches upstream install.sh), connected to `dokploy-network`.
- Coexists with the standalone `nextcloud-aio` container — Swarm only manages services it created.

## Management

```bash
# Swarm services
docker service ls
docker service ps dokploy
docker service logs -f dokploy

# Standalone Traefik
docker ps --filter name=dokploy-traefik
docker logs -f dokploy-traefik

# Postgres secret
docker secret ls

# Upgrade Dokploy (re-run with a new tag)
ansible-playbook automation.yml --tags dokploy -e dokploy_version=v0.27.0
```

## Setup

1. Install required Ansible collection: `ansible-galaxy collection install -r requirements.yml`
2. Deploy the role: `ansible-playbook automation.yml --tags dokploy`
3. Browse to `http://<server-ip>:3000` and complete the Dokploy admin setup wizard.

## Cloudflare tunnel integration

Recommended pattern (no router port-forwarding needed):

1. In your Cloudflare tunnel config, add a hostname for the admin UI pointing at `http://localhost:3000`.
2. Add a wildcard hostname (e.g. `*.apps.example.com`) for deployed apps pointing at `http://localhost:80` — Traefik routes by `Host:` header to the right service.
3. In Dokploy, disable Let's Encrypt and serve apps over plain HTTP. Cloudflare terminates TLS at the edge.
4. Keep router ports 80/443 closed.

## Idempotency

- Re-running the role does **not** tear down Swarm or the network (unlike `install.sh`).
- `docker_swarm`, `docker_network`, `docker_swarm_service`, and `docker_container` all reconcile in-place; image-tag changes trigger Swarm rolling updates.
- The postgres password is generated once on first run and persisted to `/etc/dokploy/.postgres_password`. To rotate, delete that file and `docker secret rm dokploy_postgres_password`, then re-run.

## Notes

- Requires `community.docker` Ansible collection (declared in `requirements.yml`).
- Docker is a hard requirement on the host — already present because of the `nextcloud-aio` role.
- Dokploy needs `/var/run/docker.sock` mounted into its container; this is what gives it the ability to deploy other services on your Swarm.
